# codex-rs/core/tests/suite/snapshots/

Insta snapshot files for integration test assertions.

## What this folder does

Contains `.snap` files generated by the `insta` snapshot testing library. These snapshots capture complex expected outputs for integration tests, ensuring that changes to model-visible layouts, compaction results, and other structured outputs are intentional.

## Naming convention

Snapshot files follow the pattern:
```
all__suite__<module>__<test_name>.snap
```

## Key snapshot categories

| Category | Tests |
|----------|-------|
| `compact__*` | Context compaction output shapes (manual, mid-turn, pre-turn) |
| `compact_remote__*` | Remote compaction with realtime start/end restatement |
| `compact_resume_fork__*` | Compaction behavior during resume/fork (rollback past compaction) |
| `model_visible_layout__*` | What the model sees in context (environment, agents, overrides, personalities) |

## Where it plugs into

- **Generated by**: Integration tests in `tests/suite/` using `insta::assert_snapshot!()` macros
- **Updated via**: `cargo insta review` when outputs intentionally change
- **CI**: Snapshot mismatches cause test failures, ensuring review of any layout changes
