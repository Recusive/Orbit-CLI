# codex-rs/codex-backend-openapi-models/

Auto-generated OpenAPI model types for the Codex backend API.

## What this folder does

Contains Rust struct definitions generated from the Codex backend OpenAPI specification. These models represent API request and response payloads for tasks, rate limits, configuration, and pull requests. The code is machine-generated and should be regenerated (not hand-edited) when the API changes.

## Where it plugs in

- Used by `codex-backend-client` which re-exports and wraps these types
- Generated by a code generation script (not included in this crate)

## Imports from

- `serde` / `serde_json` -- JSON serialization/deserialization
- `serde_with` -- advanced serde helpers

## Exports to

All types exported via `pub mod models` from `lib.rs`:

- `RateLimitStatusPayload` / `RateLimitStatusDetails` / `RateLimitWindowSnapshot` -- rate limit response models
- `CreditStatusDetails` / `AdditionalRateLimitDetails` -- credits and additional rate limits
- `PlanType` -- account plan type enum
- `PaginatedListTaskListItem` / `TaskListItem` -- task listing
- `CodeTaskDetailsResponse` / `TaskResponse` -- task detail responses
- `ConfigFileResponse` -- configuration file response
- `ExternalPullRequestResponse` / `GitPullRequest` -- PR-related models

## Key files

| File | Role |
|------|------|
| `Cargo.toml` | Crate manifest; allows `clippy::unwrap_used` and `clippy::expect_used` for generated code |
| `src/lib.rs` | Re-exports `models` module; suppresses lint warnings for generated code |
| `src/models/mod.rs` | Generated module aggregator |
| `src/models/*.rs` | Individual generated model files (one per API type) |
