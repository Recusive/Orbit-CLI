# codex-rs/tui/src/onboarding/snapshots/

Insta test snapshots for the onboarding module.

## What this folder does

Contains `.snap` files generated by the `insta` crate during test runs. These snapshots capture the expected rendered output of onboarding components, primarily the trust directory widget (e.g., rendering for git repositories).

## What it plugs into

- **../trust_directory.rs**: Tests in the trust_directory module use `insta::assert_snapshot!()` to verify rendered output against these snapshot files.
- Snapshot files follow the naming pattern `codex_tui__onboarding__trust_directory__tests__*.snap`.
