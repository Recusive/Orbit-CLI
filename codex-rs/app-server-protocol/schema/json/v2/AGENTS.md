## Directory Map
The summary below is based on the existing directory documentation and cross-checked against the files currently present here.

### Purpose

Auto-generated JSON Schema files for the v2 protocol types. This directory contains hundreds of individual `.json` files, one per v2 protocol type, covering the full breadth of the app-server API.

### Contents

Each file is named after its corresponding Rust type (e.g., `ThreadStartParams.json`, `TurnCompletedNotification.json`, `AccountUpdatedNotification.json`). The files define the JSON Schema for:

- Thread lifecycle types (start, resume, fork, read, list, archive, unarchive, rollback, etc.)
- Turn management types (start, interrupt, steer, completed, etc.)
- Config read/write types
- Plugin management types (install, list, read, uninstall)
- Account and auth types
- Filesystem operation types
- MCP server types
- Command execution types
- Model listing types
- Notification types (agent message delta, plan delta, token usage, etc.)
- Approval request/response types
- And many more

### Regeneration

Generated by `cargo run --bin write-schema-fixtures` or `just write-app-server-schema`. Do not edit manually.
