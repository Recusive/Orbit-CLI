## Directory Map
The summary below is based on the existing directory documentation and cross-checked against the files currently present here.

### Purpose

Auto-generated JSON Schema fixtures for the app-server protocol. Contains individual `.json` files for each protocol type plus consolidated schema bundles.

### Structure

- **Root files** -- Consolidated schema bundles (`codex_app_server_protocol.schemas.json`, `codex_app_server_protocol.v2.schemas.json`) and individual type schemas shared across versions (e.g., `ClientRequest.json`, `ServerNotification.json`, `JSONRPCMessage.json`).
- **`v1/`** -- JSON Schema files for v1 protocol types (e.g., `InitializeParams.json`, `InitializeResponse.json`).
- **`v2/`** -- JSON Schema files for all v2 protocol types, one file per type (hundreds of files covering threads, turns, config, plugins, accounts, etc.).

### Regeneration

Generated by `cargo run --bin write-schema-fixtures` or `just write-app-server-schema`. Do not edit these files manually.

- Used by external tooling for JSON Schema validation of app-server messages.
- Validated by `tests/schema_fixtures.rs` to ensure they stay in sync with Rust types.
