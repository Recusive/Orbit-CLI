## Directory Map
The summary below is based on the existing directory documentation and cross-checked against the files currently present here.

### Purpose

Auto-generated TypeScript type definitions for all v2 protocol types. This is the largest subdirectory of the TypeScript schema fixtures, containing hundreds of `.ts` files -- one per v2 protocol type.

### Contents

Each file exports a single TypeScript type or interface corresponding to a Rust v2 protocol struct or enum. Categories include:

- **Thread types** -- `Thread.ts`, `ThreadStartParams.ts`, `ThreadStartResponse.ts`, `ThreadItem.ts`, `Turn.ts`, etc.
- **Notification types** -- `TurnCompletedNotification.ts`, `AgentMessageDeltaNotification.ts`, `ThreadStatusChangedNotification.ts`, etc.
- **Config types** -- `Config.ts`, `ConfigReadParams.ts`, `ConfigWriteResponse.ts`, etc.
- **Plugin types** -- `PluginInstallParams.ts`, `PluginListResponse.ts`, `PluginDetail.ts`, etc.
- **Account types** -- `Account.ts`, `GetAccountResponse.ts`, `LoginAccountParams.ts`, etc.
- **Approval types** -- `CommandExecutionRequestApprovalParams.ts`, `FileChangeRequestApprovalParams.ts`, etc.
- **Model types** -- `Model.ts`, `ModelListParams.ts`, `ModelListResponse.ts`, etc.
- **MCP types** -- `McpServerStatus.ts`, `McpServerElicitationRequestParams.ts`, etc.
- **Filesystem types** -- `FsReadFileParams.ts`, `FsWriteFileParams.ts`, etc.
- **And many more** -- Skills, commands, reviews, plans, rate limits, etc.

An `index.ts` barrel file re-exports all types from this directory.

### Regeneration

Generated by `cargo run --bin write-schema-fixtures` or `just write-app-server-schema`. Do not edit manually.

### What It Plugs Into

- Imported by TypeScript clients via the parent `typescript/index.ts` barrel.
- Part of the TypeScript schema fixture set validated by `tests/schema_fixtures.rs`.

### Exports To

- TypeScript/JavaScript consumers that need typed v2 protocol interfaces.
