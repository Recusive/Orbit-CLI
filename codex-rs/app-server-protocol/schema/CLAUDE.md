# app-server-protocol/schema

## Purpose

Root directory for generated schema fixtures that define the app-server protocol contract in language-agnostic formats. These files are auto-generated from Rust types and committed to version control so that cross-language consumers (TypeScript clients, documentation tools, etc.) can consume the protocol definitions without depending on the Rust build.

## Structure

- **`json/`** -- JSON Schema definitions for all protocol types, organized into v1 and v2 subdirectories.
- **`typescript/`** -- TypeScript type definitions generated from the Rust types, organized into subdirectories matching the protocol namespace.

## Regeneration

Fixtures are regenerated by running:

```bash
just write-app-server-schema
```

Or directly via the binary:

```bash
cargo run --bin write-schema-fixtures -- --schema-root schema/
```

The `tests/schema_fixtures.rs` test verifies that committed fixtures match the freshly generated output. If the test fails, run the regeneration command above.

## What It Plugs Into

- Consumed by TypeScript/JavaScript clients that need typed protocol definitions.
- Validated by `tests/schema_fixtures.rs` in CI to ensure fixtures stay in sync with Rust source types.
- Generated by `src/export.rs` and `src/schema_fixtures.rs`.

## Exports To

- TypeScript consumers import from `schema/typescript/` for typed client/server message types.
- JSON Schema consumers use `schema/json/` for validation and code generation.
